my golang learning journey

Books:
John Bodner - Learning Go (First edition)
A. Donowan, B. Kerighan - The Go Programming Languege 



Learning Plan:

### 1. **Основи Go:**
    - Синтаксис, Типи Даних, Змінні, Оператори.
    - Керуючі Конструкції (Цикли, Розгалуження).
    - Функції, Методи, Структури.
    - Масиви, Зрізи, Карти.
    - Робота з Рядками.
    - **Обробка Помилок.**
    - Пакети та Імпорт.
    - **"Enum" у Go** (константи, `iota`, власні типи).

### 2. **Конкурентність (Goroutines та Channels):**
    - Основи Goroutines та Channels.
    - Буферизовані та Небуферизовані Канали.
    - `sync.WaitGroup` для синхронізації Goroutines.
    - `select` для неблокуючих операцій з каналами та таймаутів.
    - Паттерни Fan-Out/Fan-In.
    - Практичні завдання та експерименти.

---

### 3. **Робота з Файловою Системою (Локальною та Мережевою):**
    - Відкриття, Читання, Запис, Закриття Файлів.
    - Створення, Видалення, Перейменування Файлів та Директорій.
    - Отримання Інформації про Файли та Директорії (розмір, час модифікації, права).
    - **Копіювання Файлів (збереження атрибутів, прав).**
    - Абсолютні та Відносні Шляхи.
    - **Робота з UNC шляхами (Windows).**
    - **FTP (File Transfer Protocol):**
        - Підключення, Завантаження (Upload), Вивантаження (Download) Файлів.
        - Перегляд Директорій.
        - Використання сторонніх Go-бібліотек для FTP.
    - **SMB/CIFS (Server Message Block / Common Internet File System):**
        - Огляд Концепцій та Складнощів.
        - Використання сторонніх Go-бібліотек для взаємодії з SMB-ресурсами.
        - Підключення до мережевих ресурсів, читання/запис.
    - **S3 (Amazon Simple Storage Service):**
        - Концепції Об'єктного Сховища та S3.
        - Налаштування AWS SDK for Go v2.
        - Конфігурація Автентифікації та Регіонів.
        - Основні Операції (Upload, Download, Delete, List Objects).
        - Попередньо Підписані URL (Pre-signed URLs) (опціонально).

---

### 4. **Архівування та Розархівування Даних:**
    - Використання пакету `archive/zip` для роботи з ZIP-архівами.
    - Використання пакету `archive/tar` для роботи з TAR-архівами.
    - Створення Архівів.
    - Розархівування Даних.
    - Обробка Великих Файлів.

---

### 5. **Псевдо-CLI (Інтерфейс Командного Рядка):**
    - Використання пакету `flag` для обробки аргументів командного рядка.
    - Використання пакету `os` для взаємодії з операційною системою.
    - **Форматування Виводу в Консоль (таблиці, кольори, тощо).**
    - Створення Інтерактивного CLI (введення даних користувачем).
    - Обробка Помилок та Вивід Повідомлень Користувачу.
    - **Розширений CLI (опціонально):**
        - Використання популярних CLI-фреймворків (наприклад, Cobra, spf13/Viper).

---

### 6. **Запуск Додатку в Режимі CLI або Web:**
    - Використання пакету `flag` для визначення режиму запуску.
    - Умовна Компіляція (build tags) для розділення коду CLI та Web (за потреби).
    - **Веб-сервери та Веб-сайти (основи):**
        - Використання пакету `net/http` для створення базового веб-сервера.
        - Обробка HTTP-запитів та Відповідей.
        - Маршрутизація (routing) з `net/http` та/або сторонніми роутерами (наприклад, `gorilla/mux`).
        - Робота з HTML-шаблонами (`html/template`).
        - Обробка Форм та Статичних Файлів.
        - **Інтеграція HTMX:**
            - Принципи HTMX (HTML over the wire).
            - Додавання HTMX до HTML-шаблонів.
            - Обробка HTMX-запитів на Go-сервері.
            - Повернення Часткових HTML-відповідей.
            - Приклади: динамічні списки, оновлення частин сторінки, сабміт форм без повного перезавантаження.

---

### 7. **Розробка API (REST):**
    - Принципи RESTful Архітектури.
    - Створення RESTful Ендпоінтів.
    - Обробка Різних HTTP-методів (GET, POST, PUT, DELETE).
    - JSON-серіалізація/десеріалізація (`encoding/json`).
    - Обробка Помилок у REST API (статусні коди HTTP).
    - Міжпроцесна Взаємодія (CORS).
    - (Опціонально) Використання популярних веб-фреймворків для API (Gin, Echo, Fiber).

---

### 8. **Робота з Базами Даних:**
    - **Реляційні Бази Даних (SQL):**
        - Взаємодія з базою даних через `database/sql` пакет.
        - Підключення до різних баз даних (PostgreSQL, MySQL, SQLite) за допомогою драйверів.
        - Виконання SQL-запитів (SELECT, INSERT, UPDATE, DELETE).
        - Підготовлені Оператори (Prepared Statements).
        - Транзакції.
        - (Опціонально) Використання ORM/Query Builders (наприклад, GORM, `sqlx`).
    - **NoSQL Бази Даних:**
        - Огляд Різних Типів NoSQL БД (документо-орієнтовані, key-value, графові).
        - Взаємодія з популярними NoSQL базами даних (наприклад, MongoDB, Redis) за допомогою їхніх офіційних Go-клієнтів.
    - **Кешуючі Бази Даних (in-memory, Redis):**
        - Концепції Кешування.
        - Використання Redis як кешу.
        - Операції з Даними (SET, GET, HSET, HMGET).

---

### 9. **Робота зі Службами Черг:**
    - Концепція Черг Повідомлень (Message Queues).
    - Основні Сценарії Використання (фонові завдання, розподілені системи).
    - Взаємодія з популярними брокерами повідомлень (наприклад, RabbitMQ, Kafka) за допомогою їхніх офіційних Go-клієнтів.
    - Відправлення та Отримання Повідомлень.
    - Обробка Підтверджень та Помилок.

---

### 10. **Просунута Робота з Мережею та Безпекою:**
    - **Мережеві Шари та Протоколи (поглиблено):**
        - Детальніше про TCP/IP стек.
        - Основи Сокетного Програмування (`net` пакет): TCP-сервер/клієнт, UDP-сервер/клієнт.
        - Парсинг Мережевих Пакетів (використання бібліотек типу `gopacket`).
        - **Сніфінг Трафіку:** Базові концепції та реалізація простих сніферів (наприклад, для моніторингу конкретних портів або протоколів).
    - **Захист від Мережевих Атак (концепції та основи реалізації):**
        - Що таке DDoS-атаки та їхні типи.
        - **Механізми Блокування по IP/ASN:**
            - Отримання IP-адреси клієнта.
            - Використання баз даних IP-адрес для визначення ASN (Autonomous System Number) та географії.
            - Приклади реалізації простих блеклістів/вайтлістів за IP.
            - (Важливо: Створення повноцінної системи захисту від DDoS — це дуже складно, але ми розглянемо її архітектурні принципи та базові механізми).
        - Кросплатформність: Як Go допомагає писати код, що працює на різних ОС.

---

### 11. **Власні Планувальники Завдань (Custom Scheduler):**
    - Концепції Планування (scheduling).
    - Реалізація Власного Планувальника Завдань у Go:
        - Зберігання Завдань (в пам'яті, з можливістю персистентності).
        - Визначення Розкладу (одноразово, повторювано, за інтервалом, за розкладом Cron).
        - Виконання Завдань (запуск Goroutines або зовнішніх скриптів/команд).
        - Управління Життєвим Циклом Завдань (зупинка, видалення).
        - Обробка Помилок у Завданнях.
        - (Це буде спрощена версія, щоб зрозуміти принципи, схожі на Windows Scheduler/Cron).

---

### 12. **Логування та Моніторинг:**
    - **Логування:**
        - Використання стандартного пакету `log`.
        - Використання сторонніх бібліотек для логування (наприклад, `logrus`, `zap`) для структурованого логування.
        - Логування в Консоль.
        - **Логування у Файл (з ротацією, якщо можливо).**
        - Рівні Логування (DEBUG, INFO, WARN, ERROR).
        - Обробка Помилок Логування.
    - **Моніторинг з Prometheus та Grafana:**
        - Концепції метрик, time-series баз даних.
        - **Prometheus:** Go-клієнт для експонування власних метрик.
        - **Grafana:** Візуалізація Prometheus-метрик (базове налаштування та створення дашбордів).
    - **Централізоване Логування з ELK Stack (Elasticsearch, Logstash, Kibana):**
        - Концепції централізованого логування.
        - Надсилання логів Go-додатків до Elasticsearch (наприклад, через Logstash або прямий клієнт).
        - Базовий пошук логів та візуалізація в Kibana.

---

### 13. **Тестування та Якість Коду:**
    - **Базове Тестування в Go:**
        - Юніт-тести (`go test`).
        - Табличні тести (Table Driven Tests).
        - Заглушки (mocks), стаби (stubs) та фейки (fakes).
        - Тестування HTTP-хендлерів.
    - **Тестування Навантаження (Benchmark testing):**
        - Використання `go test -bench`.
    - **Тестування з Конкурентністю:**
        - Тестування Goroutines та Channels.
    - **Покриття Коду Тестами (Code Coverage):**
        - Використання `go test -cover`.
        - Аналіз Звітів про Покриття (HTML-звіти).
        - Важливість Покриття та його Обмеження.
    - **Інтеграційні та E2E (End-to-End) Тести (концепції).**

---

### 14. **Структура та Дизайн Проектів Go:**
    - **Структурування Проектів:**
        - Стандартні Конвенції Go (Go Project Layout).
        - Розділення Коду на Пакети.
        - Розподіл Функціоналу (domain, service, repository, handler).
        - Організація Тегів та Версіонування.
    - **Документування Проектів:**
        - Важливість Документації.
        - Використання `go doc` для документації коду.
        - Написання Коментарів у Go.
        - Створення файлів `README.md`.
    - **Дизайн Проектів (Архітектура):**
        - Концепції Архітектури (рівні, компоненти).
        - **Топ-Левел Архітектура (High-Level Design).**
        - **Декомпозиція до Більш Конкретних Задач (Low-Level Design).**
        - Шаблони Проектування (Design Patterns) у Go (адаптер, стратегія, будівельник тощо).
        - Принципи SOLID.

---

### 15. **Розгортання Додатків (Deployment) та CI/CD:**
    - **Збірка Go-Додатків:**
        - Статичні Бінарники, Крос-компіляція.
        - Оптимізація Розміру Бінарників.
    - **Контейнеризація з Docker:**
        - Написання Ефективних `Dockerfile` для Go-додатків.
        - Багатоетапні збірки (Multi-stage builds).
        - Docker Compose для локальних середовищ розробки.
    - **Оркестрація Контейнерів з Kubernetes:**
        - Базові Концепції Kubernetes (Pods, Deployments, Services, Namespaces).
        - Розгортання Go-додатків у Kubernetes.
        - **Масштабування:** Горизонтальне Автомасштабування Подів (HPA) для Go-додатків.
        - Базове виявлення сервісів та балансування навантаження в Kubernetes.
    - **CI/CD Пайплайни (Continuous Integration / Continuous Deployment):**
        - **GitHub Actions / GitLab CI/CD:**
            - Введення в Принципи CI/CD.
            - Налаштування Базових Пайплайнів Збірки, Тестування та Деплою для Go-додатків.
            - Автоматизація Збірки Docker-образів та їх Пушу до реєстру.
            - Автоматизація Розгортань у Kubernetes.
            - Розгортання для Конкретних Середовищ.
